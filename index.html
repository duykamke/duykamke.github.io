Smart Toy Car with Object Detection

Cyber physical systems and systems of systems

Project Summary
This project aims to resolve a car's autonomous handling of an intersection.
Features
  • Following leading car ACC
    The toy car can follow a leading toy car by using either object detection or shape/color recognition. It has a function to accelerate or decelerate based on the size of the leading car, including a fail break when no car is detected.
  • Safely stop
    The toy car can also detect a stop sign and will stop when the stop sign grows to a certain size in the car's camera. Of course, the car in front has to leave the intersection first before the toy car can stop at the stop line.
  • Determine when to go
    We made use of object detection to count the other toy cars in the intersection. The car has a queueing system that can determine when to go after all the cars in queue have left the intersection. Object detection is also used to detect the other cars leaving the intersection.
  • Receive direction wirelessly
    The car is able to receive direction wirelessly (eg. turn left/straight/right) and act accordingly once it's ready in queue at the intersection.
  • Check traffic rule for direction
    We used shape/color recognition using the car's camera to detect traffic signs (eg. can't turn left/go straight/turn right). When the car has received direction wirelessly, it will choose to act accordingly to both the command and the traffic signs.

Text to MBTI

Project Summary
The system is a personality predictor based on the MBTI personality types: given a sentence or text input by the user (ideally associated to their values, beliefs, or simply related to them somehow) the system will predict to which personality type the text is mostly related with.
Features
  • As a user you would start the web application and be given a text field where you can provide a post, sentence or paragraph as an input, the longer and more context filled the input is the more precise the prediction will most likely be, but it can accept single word inputs as well. The input should be in english and use the latin alphabet, it will take in other characters from other alphabets but it will ignore them when doing the prediction.
  • Once you have provided an input you will be given a result, the result will be showing you the type it predicted the input to have, how sure the it is about each character in the character and provide a description of people who has that personality type and also show examples of famous people who are known to have the same personality type.
  • As an admin you are able to retrain the model with a new CSV file if valid data is provided in it, the same data can also be used to re-evaluate the interference models before hand to see if it will actually provide a benefit to the system.
